#プログラムとメモリ

プログラムは実行時にまずメモリにロードされる。
そして、メモリからCPUがデータを読み出すことによってプログラムは実行される。
したがって変数などもすべてメモリ上に置かれている。
メモリ上の位置はOSにより管理されており、アドレスと呼ばれている。
このアドレスを操作する方法がC++では提供されている。
それがポインタだ。

#ポインタ演算
ポインタの記法はややこしいので、
概念を把握するのとポインタの記法を理解しようとするのは難しいと思う。
そこでとりあえず、(int型)変数のアドレスを取得する関数peekと与えられたアドレスの領域に値を書き込むpoke関数を用意した。
これらの関数を用いてアドレスを用いた操作に慣れよう。

```
#include<iostream>

int* peek(int& var){
    return &var;
}

void poke(int* ptr, int val){
    *ptr = val;
}

int main(){

    int a = 2;
    int b = 5;

    std::cout <<"adrees of a:" << peek(a) << ", value of a:" << a << std::endl;

    //アドレスから書き換え
    poke(peek(a), 4);

    std::cout <<"adrees of a:" << peek(a) << ", value of a:" << a << std::endl;

    std::cout <<"adrees of b:" << peek(b) << ", value of b:" << b << std::endl;
    std::cout <<"adrees of a-b:" << peek(a) - peek(b) << std::endl;

    //アドレスの差
    int differential = peek(b) - peek(a);

    //aのアドレス+(bとaのアドレスの差) = bのアドレス
    poke(peek(a) + differential, 3);


    std::cout <<"adrees of b:" << peek(b) << ", a value:" << b << std::endl;

    return 0;
}
```

している操作は

1. アドレスを取得する
2. アドレスから変数に戻す(参照外し)
3. アドレスの値をすすめる(or戻す)

という3つのことだ。これをつかうことによってaのアドレスからすすめてbの値を書き換えるようなことまで出来てしまう。

これをpeek,pokeを使わずにポインタの文法を使って書きなおそう。

```
#include<iostream>

int main(){

    int a = 2;
    int b = 5;

    std::cout <<"adrees of a:" << &a << ", value of a:" << a << std::endl;

    //アドレスから書き換え
    *(&a) = 4;

    std::cout <<"adrees of a:" << &a << ", value of a:" << a << std::endl;

    std::cout <<"adrees of b:" << &b << ", value of b:" << b << std::endl;
    std::cout <<"adrees of a-b:" << &a - &b << std::endl;

    //アドレスの差
    int differential = &b - &a;

    //aのアドレス+(bとaのアドレスの差) = bのアドレス
    *(&a + differential) = 3;

    std::cout <<"adrees of b:" << &b << ", value of b:" << b << std::endl;

    return 0;
}
```

見た目は気持ち悪くなったかもしれないがやっていることとしては同一だ。
ポイントは

- &(変数)=変数のアドレス
- *(変数のアドレス)=変数

ということだ。
ちょうど&と*は逆関数のような関係になっている。

これだけでも便利だがアドレスを保存する変数(あるいは型)があったほうがさらに便利になるだろう。
このアドレスを保存する変数はポインタと呼ばれている。
その名前が指す通り(アドレスを用いて)変数を間接的に指し示す。
レーザーポインタ使う際はレーザーポインタ自身ではなくレーザーが指し示す先が重要だろう。
プログラミングでのポインタも指示している先(変数など)が重要だ。とりあえずサンプルを示そう。
やっていることはポインタに変数のアドレスを代入して、ポインタ経由で操作を行っているか否かなので前のサンプルと本質は変わらない。
変数の中身に何が入っているかを意識してコードを読み書きしてほしい。 

```
#include<iostream>
  
int main(){
  
    int HP=4;
 
    //int*というintに対するポインタ型の変数ptrを宣言
    int* ptr;
    //変数HPのアドレスを代入する
    ptr=&HP;
 
    std::cout<<"HP:"<< HP <<std::endl;
    //これはptrにHPのアドレスが入っているので「*(&HP)」と等価
    std::cout<<"*ptr:"<< *ptr <<std::endl;
     
    //*(&HP)=3;と等価(HPに3が代入される。)
    *ptr=3;
    std::cout<<"HP:"<< HP <<std::endl;
     
    return 0;
}
```

一般にポインタの宣言は以下のようになる。

```
型名* ポインタ名;
```

ここで注意してほしいのがポインタの宣言の時に使われる*と参照外し時の*の意味は全然違うという点だ。 
何でこんなわかりにくいことになっているのかは知らないが文法でそう規定されているのだから認めるしかない。
ここで再度ポインタに関する操作をまとめてみよう。
また、よくアドレスの意味でポインタが使われることもあるということもここで伝えておく。

1. アドレス(ポインタ)を取得する
2. ドレスを進める(or戻す)
3. 参照を外す(変数に戻す)
4. アドレス自体を変数に入れる
 
いろいろなところで複雑でわかりにくいと呼ばれるポインタだが、このように実際にはポインタを使ってする操作というのは基本的にこの4つしかない。 あとはこれらとメモリの知識を用いて色々な操作をすれば良い。 また、記号がややこしいかもしれないが記号は記号でしかない。 迷ったら上の4つのことを思い出して欲しい。
特に注意してほしいのが*をつけている時とつけていない時のポインタの振る舞いだ。 これは例えば次のような例を考えてみるとイメージがつかめるかもしれない。
当たっている宝くじをお金に交換してそのお金を自動販売機に入れればジュースが買える。
しかし宝くじ自体を自販機に入れても買えないだろう。 

#関数とポインタ
今までは「ポインタがどういうものなのか」という説明だったのであまり実用性は感じられなかったかもしれない。
以下ではポインタとメモリの実用例をいくつかみていこう。
まずは課題で何回か出てきた変数の値を入れ替えるという処理をswapという関数するというものだ。
とりあえず、変数を入れ替えることを意図して以下のようなプログラムを考えた。 

```
#include<iostream>
 
void swap(int a,int b){
    int tmp=a;
    a=b;
    b=tmp;
    return;
}
 
int main(){
     
    int a=5;
    int b=1;
     
    swap(a,b);
 
    std::cout<<a<<","<<b<<std::endl;
 
     
}
```

実行してみれば(しなくても)わかるようにこれは意図した動作をしない。
これを解決するためにポインタを利用するのだ。
swap関数をここで実装してもいいのだが、演習に回すとしてサンプルではmain関数の変数のポインタを渡して常に2に書き換える関数を実装してみよう。
関数の引数としてポインタをうけとれば先ほどの議論からスコープ的には許されていなくても ポインタが指し示している先の値を書き換えることができる。 

```
#include<iostream>
 
//2に書き換える関数
void rewrite(int* ptr){
    *ptr=2;
    return;
}
 
int main(){
  
    int HP=4;
    //&を忘れない!
    rewrite(&HP);
 
    std::cout<<HP<<std::endl;
 
    return 0;
}
```

プログラムを実行してみると確かに書き換わっていることがわかる。
これは渡されるのはアドレス、つまりただの数字なのでプログラムはアドレスがさしている先のスコープを知らない。 したがって合法的に書き換えられるというわけだ。

また、ポインタのサイズはどの型でも一定で一般にアドレスの長さである。 一方例えばstd::string型はどんな長さのサイズの文字列も持つことができる。 したがって文字列を関数に渡すとそこでコピーが発生するので数百MBのコピーなどが発生する可能性がある。 そのため一般にポインタ渡しにしたほうが良い。 

#配列とポインタ
次に配列の仕組みを見てみることにしよう。

まず大事なことは配列名だけ(array)の時とarray[0]のアドレスが同じということだ。 
また配列はメモリ上に連続に確保される(規格で保証されている)。
つまり配列は単なるメモリ上に連続に並んだ変数でしかないのでhoge[n]==*(hoge+n)となる。(実際にこのように[]が定義されている。)
したがって 2[hoge] というコードでも全く問題なくコンパイルが通るし実行できる。 (もちろんこんなわかりにくいプログラムは勧められないが...)
(ちなみにポインタも同じように[]で参照外しができる。)
以上のサンプルとしては以下のような感じだ。 

```
#include<iostream>
 
int main(){
     
    int array[4];
    for(int i=0;i<4;++i){
        array[i]=i; 
    }

    std::cout<<array<< "," << &array[0] <<std::endl;
     
    std::cout<<*array<<std::endl;
    std::cout<<*(array+1)<<std::endl;
    std::cout<<2[array]<<std::endl;

    int a;
    int* a_ptr = &a;
    std::cout<<a[0]<<std::endl;
    std::cout<<0[a]<<std::endl;
     
}
```
